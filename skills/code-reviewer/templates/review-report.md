# Code Review Report Template

## Summary

One paragraph: scope (files, lines), overall quality, key findings.

## Critical Issues

Bugs, security vulnerabilities, breaking changes. Must fix before commit.

- **Title** (`file:line`)
  - **Problem**: what's wrong
  - **Impact**: why it's critical
  - **Fix**: concrete solution (code snippet if short)

## Important Suggestions

Performance, maintainability, significant improvements. Should address.

- **Title** (`file:line`)
  - **Observation**: what was noticed
  - **Recommendation**: proposed change with rationale

## Minor Improvements

Style, docs, minor optimizations. Nice to have. One line each.

## Positive Highlights

Acknowledge good practices with file:line references.

## Next Steps

Prioritized action list. Include review status: Approved with changes / Needs revision / Blocked.
